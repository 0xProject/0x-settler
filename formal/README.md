# Formal Verification

Machine-checked Lean 4 correctness proofs for root math libraries in 0x Settler. Zero `sorry`, no axioms beyond the Lean kernel.

## Scope

| Proof | Solidity source | What is proved |
|-------|----------------|----------------|
| `sqrt/SqrtProof` | `src/vendor/Sqrt.sol` | `_sqrt`, `sqrt`, `sqrtUp` correct on uint256 |
| `sqrt/Sqrt512Proof` | `src/utils/512Math.sol` | `_sqrt` (512-bit) correct: `sqrt(x_hi * 2^256 + x_lo) = natSqrt(x)` |
| `cbrt/CbrtProof` | `src/vendor/Cbrt.sol` | `_cbrt`, `cbrt`, `cbrtUp` correct on uint256 |
| `cbrt/Cbrt512Proof` | `src/utils/512Math.sol` | `_cbrt` (512-bit) correct: `cbrt(x_hi * 2^256 + x_lo) = icbrt(x)` |
| `ln/LnProof` | `src/vendor/Ln.sol` | `lnWadToRay`, `lnWad` monotonicity and floor/cut specifications |

## Method

1. **Algebraic lemmas** prove one-step safety and correction logic (Babylonian / Newton-Raphson steps).
2. **Finite domain certificates** (auto-generated by Python scripts) cover all uint256 octaves with `by decide` proofs.
3. **The `formal/yul` Lake importer** consumes `forge inspect ... ir` output and emits ignored `*Yul.lean` model modules.
4. **End-to-end bridge theorems** prove the generated EVM models equal the hand-written mathematical specs.

## Build

All generated files (`SqrtYul.lean`, `FiniteCert.lean`, `LnYul.lean`, `FloorCertLit.lean`, etc.) are `.gitignore`d and regenerated in CI. See `.github/workflows/*-formal.yml` for the canonical build steps.

Build the shared importer once before generating model modules:

```bash
lake -d formal/yul build yul_importer
```

```bash
# --- 256-bit sqrt ---
forge inspect src/wrappers/SqrtWrapper.sol:SqrtWrapper ir | \
  lake -d formal/yul exe yul_importer --kind sqrt \
    --output formal/sqrt/SqrtProof/SqrtProof/SqrtYul.lean

python3 -m formal.python.sqrt.generate_sqrt_cert \
  --output formal/sqrt/SqrtProof/SqrtProof/FiniteCert.lean

cd formal/sqrt/SqrtProof && lake build

# --- 512-bit sqrt ---
FOUNDRY_SOLC_VERSION=0.8.34 \
  forge inspect src/wrappers/Sqrt512Wrapper.sol:Sqrt512Wrapper ir | \
  lake -d formal/yul exe yul_importer --kind sqrt512 \
    --output formal/sqrt/Sqrt512Proof/Sqrt512Proof/Sqrt512Yul.lean

# FiniteCert.lean (shared with SqrtProof) must be generated first — see above.
cd formal/sqrt/Sqrt512Proof && lake build

# --- cbrt ---
forge inspect src/wrappers/CbrtWrapper.sol:CbrtWrapper ir | \
  lake -d formal/yul exe yul_importer --kind cbrt \
    --output formal/cbrt/CbrtProof/CbrtProof/CbrtYul.lean

python3 -m formal.python.cbrt.generate_cbrt_cert \
  --output formal/cbrt/CbrtProof/CbrtProof/FiniteCert.lean

cd formal/cbrt/CbrtProof && lake build

# --- 512-bit cbrt ---
FOUNDRY_SOLC_VERSION=0.8.34 \
  forge inspect src/wrappers/Cbrt512Wrapper.sol:Cbrt512Wrapper ir | \
  lake -d formal/yul exe yul_importer --kind cbrt512 \
    --output formal/cbrt/Cbrt512Proof/Cbrt512Proof/Cbrt512Yul.lean

# FiniteCert.lean (shared with CbrtProof) must be generated first — see above.
cd formal/cbrt/Cbrt512Proof && lake build

# --- ln ---
python3 -W error -m formal.python.ln.check_ln_counterexample
python3 -W error -m formal.python.ln.check_ln_monotone

forge inspect src/wrappers/LnWrapper.sol:LnWrapper ir | \
  lake -d formal/yul exe yul_importer --kind ln \
    --output formal/ln/LnProof/LnProof/LnYul.lean

python3 -W error formal/python/ln/gen_cert_literals.py

cd formal/ln/LnProof && lake build
```
